## 🇬🇧 English

### Overview
This repository contains the core semantic planning layer for controlling a **Unitree Go2 Pro** quadruped robot using natural language. This work is part of an academic research project focused on safe Human-Robot Interaction (HRI) bridging the "embodiment gap".

Using a local Large Language Model (**Llama 3 8B**), the system processes voice-transcribed commands and translates them into safe, structured kinematic parameters using a strict JSON format and Chain-of-Thought reasoning.

### 🎥 Video Demonstration
You can watch the full, uninterrupted pipeline in action here:

[![LLM Semantic Planner for Unitree Go2](https://img.youtube.com/vi/u3mgNyLofOk/0.jpg)](https://www.youtube.com/watch?v=u3mgNyLofOk)

*Click the image above to watch the demo on YouTube.*

The video demonstrates the complete flow: from the initial verbal interaction, through the STT and LLM reasoning, to the final physical response of the robot.

### Repository Scope
This repository focuses **exclusively on the LLM prompt engineering, reasoning layer, and JSON validation software**. It includes:
* The optimized `System Prompt` with safety guardrails.
* The Python logic for LLM inference (via Ollama).
* The validation dictionary and structural filters for kinematic parameters (e.g., max velocity, duration).

### ⚠️ Important: WebRTC Middleware Dependency
This repository **does not** include the low-level communication middleware required to physically move the Go2 Pro robot. 

To transmit the validated JSON commands generated by this code to the robot, you need a WebRTC bridge. We want to give full credit to the community for this capability: the physical communication in our experiments was achieved using the reverse-engineered WebRTC interface developed by **[legion1581/unitree_webrtc_connect](https://github.com/legion1581/unitree_webrtc_connect)**. You will need to implement or run a similar bridge to test this code on real hardware.

## 🇪🇸 Español

### Resumen
Este repositorio contiene la capa principal de planificación semántica para controlar el robot cuadrúpedo **Unitree Go2 Pro** mediante lenguaje natural. Este trabajo forma parte de un proyecto de investigación académica centrado en la seguridad en la Interacción Humano-Robot (HRI) y la mitigación de alucinaciones en modelos fundacionales.

Utilizando un Modelo de Lenguaje Grande en local (**Llama 3 8B**), el sistema procesa comandos transcritos por voz y los traduce a parámetros cinemáticos estructurados y seguros mediante un formato JSON estricto y razonamiento *Chain-of-Thought*.

### Alcance del repositorio
Este repositorio se centra **exclusivamente en el diseño del prompt, la capa de razonamiento del LLM y el software de validación del JSON**. Incluye:
* El `System Prompt` optimizado con reglas de seguridad inquebrantables.
* La lógica en Python para la inferencia del LLM (vía Ollama).
* El diccionario de validación y los filtros estructurales para los parámetros cinemáticos (ej. velocidad máxima, duración).

### ⚠️ Importante: Dependencia del Middleware WebRTC
Este repositorio **no** incluye el middleware de comunicación de bajo nivel necesario para mover físicamente el robot Go2 Pro.

Para transmitir los comandos JSON validados que genera este código al robot, es necesario un puente WebRTC. Queremos dar todo el crédito a la comunidad por esta capacidad: la comunicación física en nuestros experimentos se logró utilizando la interfaz WebRTC de ingeniería inversa desarrollada por **[legion1581/unitree_webrtc_connect](https://github.com/legion1581/unitree_webrtc_connect)**. Necesitarás implementar o ejecutar un puente similar para probar este código en hardware real.
